# ADR-002: Move Self-Hosted Elastic from EC2 to Local Hardware

- Date: 2026-06-25
- Deciders: R. Santos
- Supersedes: [ADR-001](001-siem-choice.md) - the Elastic platform choice stands; only the hosting location changes.

## Status

Accepted

## Context

ADR-001 placed self-hosted Elastic on an EC2 instance in a private subnet. The lab has run successfully for almost a month, so the cloud build is proven stable. With that demonstrated, the sustained cost of running the SIEM in the cloud is no longer warranted as current estimates are ~$90-95/month due to NAT and the t3.large.

Local hardware capable of running the SIEM is already on hand.

### Decision Drivers

- Sustained-operation cost of NAT and t3.large
- Telemetry stays in AWS regardless, only where Elastic *runs* changes.
- IaC preserved — the local VM is still Terraform-provisioned (bpg/proxmox + cloud-init).

### Considered Options

- Move Elastic to a local Proxmox VM, AWS as telemetry source only — chosen
- Keep Elastic on EC2
- Schedule EC2 start/stop — rejected; adds complexity, leaves ingestion gaps, and still pays for the NAT (which cannot be stopped)

## Decision

We will move self-hosted Elastic to a local Proxmox VM. AWS is retained solely as the telemetry source. CloudTrail, GuardDuty, VPC Flow Logs, the log buckets, and the SQS queue all remain. The EC2 instance, its EBS volume, and the NAT gateway are torn down.

Filebeat on the VM pulls CloudTrail from S3 via the SQS queue. As the VM is off-AWS with no instance role, Filebeat authenticates with a dedicated least-privilege IAM user (`filebeat-local`), its keys held in the Filebeat keystore, reusing the same S3-read and SQS-read policies the EC2 role used.

The Elastic platform choice from ADR-001 is unchanged.

## Consequences

Recurring SIEM hosting cost drops to effectively zero; AWS spend falls to the usage-based telemetry sources only.

Operating the SIEM still falls on us, and the node is still single-node so a failure halts ingestion until rebuilt. The baseline now lives on local storage, so backups remain necessary. A snapshot target still needs to be configured.

A manual seam is introduced: the `filebeat-local` keys are generated by Terraform in the AWS stack but hand-carried into the keystore on the VM, since no single Terraform stack spans both AWS and Proxmox. Accepted, documented tradeoff.

`modules/siem/` (the EC2 shape) is retained but no longer invoked — available for anyone who wants the EC2 path. The NAT gateway is removed (`enable_nat = false`), leaving the private subnets isolated, which is fine as nothing runs in them now.
